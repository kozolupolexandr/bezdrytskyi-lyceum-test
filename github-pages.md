# Розміщення сайту на GitHub Pages — покрокова інструкція

Нижче наведено інструкцію, як опублікувати цей сайт на GitHub Pages. Приклад розраховано на статичний сайт (HTML/CSS/JS) або зібраний у папку з фінальними файлами.

## 1. Перевір, що сайт готовий до публікації
- Переконайся, що у корені проєкту є `index.html` або що фінальна збірка лежить у окремій папці (часто `dist` або `build`).

## 2. Створи репозиторій на GitHub
1. Увійди на GitHub.
2. Натисни `New repository`.
3. Вкажи назву репозиторію, наприклад `bezdrytskyi-lyceum`.
4. Натисни `Create repository`.

## 3. Завантаж код у репозиторій
У корені проєкту виконай:

```bash
# ініціалізація репозиторію (якщо ще не ініціалізований)
git init

# додати всі файли
git add .

# створити коміт
git commit -m "Initial commit"

# додати віддалений репозиторій (замінити URL на свій)
git remote add origin https://github.com/<your-username>/bezdrytskyi-lyceum.git

# запушити на GitHub
# якщо основна гілка main:
git branch -M main
git push -u origin main
```

## 4. Увімкни GitHub Pages
1. Відкрий репозиторій на GitHub.
2. Перейди в `Settings` → `Pages`.
3. У блоці **Build and deployment**:
   - `Source`: вибери `Deploy from a branch`.
   - `Branch`: обери `main` і `/ (root)`.
4. Натисни `Save`.

Через 1–2 хвилини з’явиться посилання на сайт у цьому ж розділі.

## 5. Якщо сайт збирається у папку (наприклад, `dist`)
- Переконайся, що збірка вже виконана й папка існує.
- У `Pages` обери гілку `main` і папку `/dist` (якщо є такий вибір) **або** зроби гілку `gh-pages`, в яку будеш пушити лише `dist`.

Приклад через `gh-pages` (для більшості збірок):

```bash
# встановити утиліту (один раз)
npm install --save-dev gh-pages

# додати скрипт у package.json
# "deploy": "gh-pages -d dist"

# зібрати сайт
npm run build

# задеплоїти
npm run deploy
```

## 6. Перевірка
Відкрий посилання, яке GitHub показує у розділі `Settings → Pages`.
